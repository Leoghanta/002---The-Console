In this workshop, we'll use the console as this is where a lot of the background communication appears and we can add javascript directly into the console.

Console.log()

The console is a good place to send messages for debugging. We can enter code directly into the console and because the browser is live, it will return values. In the console try the following code. 


console.log("Hello World");

Let's unpack this line of code.
The console is an object in the browser and we have access to the properties and methods within the console. One of the methods is log. You can see all the properties and methods for the console by just typing console. and a drop down list will appear with all the properties and methods.

log is a function, this means that it will run a piece of code in the background. It doesn't matter what that code is, but we know that the code will send a message to the console screen.

Functions have brackets. Within these brackets, we place an argument. This argument is the message we will send to the console screen. console.log("Hello World");

Lastly, we use a semi-colon. Semi-Colons are necessary in many programming languages as this tells the browser that this is the end of the statement.

Let's try some other console commands...


console.log("My Name Is ...");
console.warn("Warning, Warning");
console.error("Oh No! I'm Broken!");
console.clear();
alert("Boo!");

Problems?

Working with code is great until something goes wrong. Let's take a look at a typical error message. Try this code: 


console.log("Hello";

Not closing the brackets will provide a syntax error : Invalid or Unexpected Token. The syntax is the arangement of words and symbols that make up the code - This is the grammar that the code understands. When something is wrong and the code does not understand, the error appears. Invalid or Unexpected token meant that the code expected something (a close bracket) that wasn't there.

When the browser can not understand the code, it will stop the program and report the error. We call these exceptions.

Script.js

The console is great for testing out small snippets of code but it can only perform one operation at a time. We, as coders, would like to run a sequence of operations, and for that we will need our own program file. Typically we call this script.js but you can call this whatever you want. The suffix .js tells the browser that this is a javascript file and it contains javascript code.

This project already has a script.js and it is linked at the end of the body tag. Have a look at the HTML for this code. 


<script src="script.js"></script>

The script tag tells the browser that there is some JavaScript code and that it can be found in the source file script.js. Note that this is not a self-closing tab - this is because you could add javascript code within the open and close tags. 


<script>
    alert ("This code is running from inside the HTML page.");
</script>

This is not considered good practice so we will ensure that all our JavaScript code is placed in the correct script.js file. This file has some comments and comments which are placed between /*...*/ are ignored by the browser - this means you can use these comment tags to write your own commentary. You'll be encouraged to do this throughout the course.

Add the following code to your script.js file.


/**
* JavaScript File.
* Your code can go here
*/

alert ("I am working from inside the script.js file");
console.log ("This is a log from inside the script.js file");
console.warn ("Notice the order of operations. JavaScript works from top to bottom.");
